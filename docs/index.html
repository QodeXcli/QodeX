Why you can trust it

It doesn't trust the model. So you can.

Deterministic guardrails run around the model loop — real code, not a prompt asking nicely. This is what makes a small local model behave like a careful one.

01Syntax gate

Every edit is parsed before it's written. Broken syntax is rejected and never hits disk — the model gets the error back and retries.

02Completion gate

It can't claim "tests pass" unless a test actually ran. Unsupported "I fixed it" gets bounced back for correction.

03Auto-verify

When the model thinks it's done, QodeX runs the real checker — tsc, ruff, go vet, cargo — and force-feeds errors back.

04Git sandbox

Risky work runs on a hidden branch with checkpoints. Auto-snapshot before destructive commands, one command to undo.

05Skill scanner

Skills from GitHub are scanned for prompt injection, secret exfiltration, and destructive shell before they touch disk.

06Read-before-write

No editing an existing file from stale assumptions — a mutation is refused unless the file was actually read this session.
